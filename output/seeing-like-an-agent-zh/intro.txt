中文标题：像 Agent 一样看世界：Claude Code 的工具设计逻辑
English Title: Seeing Like an Agent: How Claude Code Designs Tools

这个视频用中文梳理了 Anthropic 关于 Claude Code 工具设计的一篇文章，重点讨论当模型主要通过工具行动时，工程团队应该如何从模型视角来设计、调整和淘汰工具。视频先提出核心问题：工具不该只是功能越多越好，而应当贴合模型当前真正会使用的能力边界。随后通过数学题的类比说明，同一项任务最适合的工具集合，取决于使用者本身的技能结构；放到 agent 身上，重点就变成模型能否理解工具、是否愿意调用、以及调用结果是否稳定。

在此基础上，视频进一步拆解了两个关键案例。第一个是 AskUserQuestion 的三次迭代，展示为什么把提问硬塞进已有流程，或者只依赖输出格式约束，都会让模型产生混乱，最终还是需要一个让模型容易理解、也让用户容易交互的独立工具。第二个是 TodoWrite 到 Task tool 的演进，说明旧工具在早期可以弥补能力缺口，但模型变强之后也可能反过来限制它的规划和多代理协作能力。后半部分则把重点放到上下文构建上，解释 Claude Code 如何从 RAG 预取上下文，演进到通过 Grep 和 Skills 让 agent 自己搜索、发现并递归读取所需信息。最后，视频用 Claude Code Guide 这个子代理案例说明，想增加能力时并不总是需要新增工具，更好的办法往往是通过渐进披露和上下文隔离，让主代理保持专注，同时把文档检索和信息压缩交给更合适的执行单元。
