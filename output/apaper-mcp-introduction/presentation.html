WHY IT MATTERS

目标不只是找到一篇论文，而是支撑连续的 paper-writing 任务链路。

apaper-mcp 要减少的，不是搜索次数，而是研究动作在工具边界之间反复被打断的成本。

当前覆盖三个关键来源，拼出一条更完整的论文发现路径

IACR ePrint

适合密码学预印本检索，也支持进一步下载 PDF 做细读。

DBLP

补全发表元数据，并可直接拉取 BibTeX 作为引用准备。

Google Scholar

做更广泛的主题发现，帮助从单点来源扩展到更大文献面。

这三个来源并不追求覆盖一切，而是先覆盖研究者最常见的“预印本 + 发表记录 + 广域发现”组合。

服务器表面很小，但已经覆盖从发现论文到收集引用的核心动作

search_iacr_papers

按主题和年份搜索 IACR ePrint，并支持细节获取。

download_iacr_paper

把 IACR 论文 PDF 保存到本地，方便后续精读和处理。

search_dblp_papers

查询发表信息、场馆与论文元数据，并可返回 BibTeX。

search_google_scholar_papers

用更广的搜索面补充相关论文发现与年份筛选。

它的集成和实现都保持得很克制，这反而让它更适合接入真实工作台

Client config

{

  "mcp": {

    "apaper-mcp": {

      "type": "local",

      "command": ["npx", "@ai4paper/apaper-mcp"]

    }

  }

}

Transport

通过 stdio 运行，便于本地 MCP client 直接启动。

Entry point

`src/index.ts` 注册工具，并用 Zod 定义输入 schema。

Platforms

`src/platforms/` 分离具体抓取与解析逻辑，避免 transport 层和平台细节混在一起。

Source differences

IACR 与 Google Scholar 走 HTML 抓取，DBLP 则使用 JSON API 并可附带 BibTeX 获取。

下一步不只是继续搜索，而是把它扩展成 paper-writing server

它的方向很明确：从 paper search tool 走向更完整的 research-writing workspace。
